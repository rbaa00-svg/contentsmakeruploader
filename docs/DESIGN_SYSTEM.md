# promo.pikkto Design System

## 목표

- 한 번의 입력으로 여러 채널용 프로모션 콘텐츠를 빠르게 검수할 수 있게 한다.
- 생성 결과를 "그럴듯한 텍스트"가 아니라 "발행 직전 초안"처럼 느끼게 만든다.
- 모바일 우선이되 데스크톱 검수 효율을 해치지 않는다.

## 디자인 원칙

### 1. Workflow First

- 가장 중요한 경험은 장식이 아니라 플로우다.
- 사용자는 입력 -> 생성 -> 검수 -> 예약을 매끄럽게 지나가야 한다.

### 2. Channel Awareness

- 모든 채널 카드를 똑같이 보이게 만들지 않는다.
- 채널별 시각 언어를 미리보기에 반영하되, 관리 UI 자체는 하나의 제품으로 읽혀야 한다.

### 3. Intentional, Not Generic

- 흔한 보라색 그라데이션, 균일한 둥근 카드 반복, 과한 중앙 정렬을 피한다.
- 제품 UI는 "콘텐츠 작업 도구"처럼 보여야지 랜딩페이지처럼 보이면 안 된다.

### 4. Editing Confidence

- 사용자가 지금 무엇을 편집 중인지 항상 명확해야 한다.
- 생성 중, 수정 중, 예약 전 상태를 시각적으로 구분한다.

## Typography

- 기본 본문 크기: 16px 이상
- 주요 화면 제목:
  - 페이지 제목 30px 전후
  - 섹션 제목 20~24px
  - 카드 타이틀 14~16px
- 긴 설명문은 line length를 제한한다.
- 제목 계층은 `h1 -> h2 -> h3` 순서를 건너뛰지 않는다.

## Spacing

- 기본 spacing scale: 4px, 8px, 12px, 16px, 24px, 32px
- 카드 내부 기본 padding: 16~24px
- 주요 섹션 간격: 24px 이상
- 검수 화면에서 에디터 컬럼과 프리뷰 컬럼은 시각적으로 구분되되 답답하지 않게 유지한다.

## Color and Surface

- 관리 UI는 중립 배경 + 명확한 primary action 조합을 사용한다.
- 상태 색상:
  - Connected / success: green 계열
  - Warning / pending: amber 계열
  - Error / failed: red 계열
- 채널 브랜드 색은 미리보기나 아이콘 accent로 제한한다.
- 정보 구조를 색으로만 구분하지 않는다.

## Interaction

- 모든 인터랙션은 hover와 focus-visible 상태를 가진다.
- 터치 타깃은 44px 이상을 권장한다.
- step 진행 상태는 현재 단계와 완료 단계를 구분해야 한다.
- destructive action은 outline/default action과 명확히 구분한다.

## Layout Rules

### Dashboard

- 연결 상태, 예약 상태, 빠른 액션이 한 화면에서 보이게 한다.
- 카드 수가 늘어나도 1열 모바일 -> 2열/3열 데스크톱으로 자연스럽게 확장한다.

### Create Flow

- 단계형 흐름을 유지한다.
- 한 번에 모든 입력을 보여주지 않는다.
- 생성 직후에는 결과보다 상태가 먼저 전달돼야 한다.

### Review Screen

- 좌측: 편집
- 우측: 미리보기
- 모바일에서는 세로 적층, 데스크톱에서는 양단 배치
- 프리뷰 탭 전환 시 현재 수정 대상과 컨텍스트를 잃지 않게 한다.

## Channel Preview Rules

- Instagram: 캡션과 이미지 중심
- Blog: 제목 1행, 본문형 읽기 경험
- X: 짧은 텍스트와 미디어 카드
- YouTube: 제목/설명 분리
- LinkedIn: 전문적 톤과 engagement 영역
- TikTok: 세로형 숏폼 경험
- Threads: 대화형 피드 문맥

## Accessibility

- 텍스트 대비를 유지한다.
- 아이콘만으로 의미를 전달하지 않는다.
- 키보드 포커스가 사라지면 안 된다.
- 날짜 선택, 탭 전환, 버튼 활성/비활성 상태를 스크린리더 친화적으로 유지한다.

## QA Checkpoints

- 모바일에서 가로 스크롤이 생기지 않는가
- 7개 채널 탭이 모두 접근 가능한가
- body text가 16px 아래로 떨어지지 않는가
- focus-visible이 사라진 요소가 없는가
- 예약 버튼, 생성 버튼, 연결 버튼의 상태 의미가 시각적으로 명확한가

## Do / Don't

- Do: 채널별 문맥 차이를 UI에 반영한다
- Do: 생성 상태와 검수 상태를 분리한다
- Do: 미리보기를 실제 발행물에 가깝게 만든다
- Don't: 전 화면 중앙 정렬
- Don't: 하나의 radius 값만 모든 컴포넌트에 반복
- Don't: 채널 관리 도구를 마케팅 랜딩처럼 꾸민다
