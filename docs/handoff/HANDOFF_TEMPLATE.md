# HANDOFF_TEMPLATE

## 목적

병렬 worktree/브랜치 작업 결과를 PM 스레드가 빠르게 검토하고 병합할 수 있도록 최소 정보를 표준화한다.

## 사용 시점

- LV 3 작업
- 병렬 브랜치 작업
- 다음 스레드나 다음 세션으로 넘겨야 하는 작업
- 병합 전에 유지/제외 대상을 명확히 해야 하는 작업

## 최소 포함 항목

- 브랜치명
- 기준 브랜치
- 핵심 목적
- 반드시 유지할 변경
- 버려도 되는 변경
- 검증 결과
- 관련 파일
- 커밋 정보

## 작성 규칙

- "무엇을 남겨야 하는지"와 "무엇은 버려도 되는지"를 분리한다.
- 검증은 실행한 항목과 결과를 함께 적는다.
- 문서 작업이면 source of truth 변경 범위를 적는다.
- 충돌 가능 파일이 있으면 경로를 명시한다.

## 템플릿

```md
[HANDOFF]
브랜치명: <branch>
기준 브랜치: <base>

핵심 목적:
- <point>

반드시 유지할 변경:
- <point>

버려도 되는 변경:
- <point 또는 없음>

관련 파일:
- <path>

검증 결과:
- <command or checklist>: <result>

메모:
- <merge note or 없음>

관련 커밋:
- <hash> <message>
```
