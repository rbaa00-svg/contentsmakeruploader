# PikkTob: med.piKKto 전용 온디바이스 AI 인퍼런스 허브
> 문서 버전: v0.9  
> 최종 수정: 2026-04-06  
> 분류: Internal / Product Strategy / Technical Specification

---

## 1. 개요

**PikkTob(픽톱)**은 med.piKKto Workbench와 연동되는 **분산형 로컬 AI 연산 모듈**이다.  
핵심 목적은 사용자의 저사양 PC 자원을 직접 소모하지 않고, 별도의 독립 하드웨어 자원에서 고사양 규제 문서 AI를 실행하는 것이다.

PikkTob은 단순한 외장형 추론 장치가 아니라 다음 가치를 제공한다.

- **Hardware Agnostic**
  - 사용자 PC가 GTX 1060급이거나 내장 그래픽 환경이어도 고난도 규제 검토를 수행할 수 있다.
- **Data Sovereignty**
  - 외부 클라우드 연결 없이 로컬 네트워크 또는 직접 연결 환경에서 의료 문서를 처리한다.
- **Workflow Integration**
  - 단순 챗봇이 아니라 문서 맥락, 연결 근거, 검토 흐름을 기준으로 실무 액션을 제공한다.
- **On-Prem / Air-Gapped Ready**
  - 인터넷과 분리된 환경에서도 med.piKKto와 함께 폐쇄형 운영이 가능하다.

---

## 2. 제품 포지셔닝

PikkTob은 med.piKKto의 부속 장치가 아니라, 다음 두 역할을 동시에 수행한다.

1. **고사양 AI 오프로딩 허브**
   - 저사양 사용자 PC 대신 문서 구조화, 정밀 검토, 보안 검사, 규정 근거 탐색을 실행한다.
2. **기업 전용 문서 보안 금고**
   - 내부 규제 문서, 검토 기록, 벡터 인덱스, 추론 로그를 외부 클라우드 없이 보관하고 처리한다.

권장 제품 문구:

- "픽톱을 연결하는 순간, 내 컴퓨터가 갑자기 더 똑똑해진다."
- "회사 문서를 외부로 보내지 않고, 사내 AI 허브에서 직접 검토한다."
- "저사양 PC에서도 Pro C 등급의 정밀 검토를 바로 호출할 수 있다."

## 3. 시스템 아키텍처

## 3.1 분산 연산 구조

PikkTob 아키텍처는 **Pikkto Client + PikkTob Engine**의 2계층 구조를 전제로 한다.

### Pikkto Client (User PC)

- med.piKKto Workbench UI/UX 렌더링
- 문서 뷰어 및 편집기
- 경량 전처리
- Basic / Pro A 등급의 로컬 실행
- 사용자 입력, 승인, 검토, 결과 확인

### PikkTob Engine (Hub)

- 고성능 CPU / GPU / NPU 기반 추론
- 대용량 메모리 및 벡터 인덱스 저장
- 규제 문서 RAG, 정밀 검토, 구조화, 보안 마스킹
- 다중 사용자 요청 큐 처리
- 로컬 업데이트 및 지식 동기화

### 연결 인터페이스

- 로컬 네트워크 (LAN / Wi-Fi)
- USB 4 / Thunderbolt 기반 직접 연결
- 향후 보안 터널링 기반 외부 접속 확장

---

## 3.2 하드웨어 적응형 레이어

PikkTob은 med.piKKto 클라이언트와 결합해 **Hardware-Adaptive Routing Layer**를 사용한다.

### Auto-Detection

- 클라이언트 실행 시 PC의 VRAM, RAM, 추론 성능, 연결된 PikkTob 존재 여부를 자동 감지한다.
- PikkTob Engine의 상태, 모델 적재 여부, 응답 가능 티어를 함께 확인한다.

### Dynamic Routing

- 저사양 PC만 존재:
  - 로컬 경량 모델로 Basic / Pro A 티어 수행
- PikkTob 연결 감지:
  - Pro B / Pro C 작업을 PikkTob Engine으로 즉시 라우팅
- 네트워크 불안정 또는 허브 미가용:
  - 로컬 최소 기능 모드로 폴백

## 4. 운영 티어 체계

### Basic (Local)

- 안내, 메타데이터 추출, 간단 요약
- 저비용 / 저지연 응답
- 로컬 단독 사용 가능

### Pro A (Local Optimized)

- 저사양 PC 환경에 맞춘 실무형 짧은 답변
- 경량 문서 분석
- 빠른 문단 수준 검토

### Pro B (PikkTob Standard)

- PikkTob 연결 시 활성화
- 문서 구조화, 연결 보조, 근거 검색, 보완점 탐지
- 다중 문서 맥락 기반의 검토 보조

### Pro C (PikkTob Premium)

- PikkTob 자원을 적극 활용하는 정밀 진단 모드
- 법적/규제 리스크 검토
- 수정안 초안 생성
- 감사 대응용 근거 정리

## 6. 5대 확장 기능

## 6.1 협업 모드 (Multi-User API)

- FastAPI 기반 로컬 API 서버
- PikkTob 한 대가 사무실 네트워크에 연결되면, 여러 사용자가 동시에 접속 가능
- 권장 동시 사용자 수: 3~5명
- 활용 예:
  - RA reviewer
  - QA approver
  - QMS 담당자
  - 외부 컨설턴트

## 6.2 자동 업데이트 (Reg-Sync)

- 새벽 시간대 델타 업데이트 스케줄러
- 식약처 / FDA / 내부 규제 문서의 신규 변경분만 동기화
- 전체 재인덱싱 대신 변경분 위주 반영으로 네트워크/스토리지 비용 최소화

## 6.3 커스텀 학습 (Local RAG Auto-Update)

- 벡터 DB 자동 갱신
- 내부 문서를 드래그 앤 드롭하면 즉시 벡터 인덱스에 반영
- 고비용 파인튜닝 대신, 기업 전용 규제 문서 메모리를 빠르게 축적

## 6.4 UI 대시보드 (Resource Monitor)

- 경량형 웹 대시보드
- CPU / GPU / NPU 점유율
- 적재된 모델 상태
- 분석 중인 문서 목록
- 요청 큐, 실패 횟수, 지연 시간

## 6.5 모바일 연동 (Secure Mobile Bridge)

- Tailscale 등 보안 터널 기반 접근
- 외부에서도 결과 확인, 승인, 간단 코멘트 가능
- 민감 문서 원문 편집보다는 승인/검토 결과 확인 중심으로 제한 운영

## 7. 모델 및 역할 분리 전략

PikkTob은 단일 모델 장비가 아니라 **역할 기반 멀티모델 허브**로 설계한다.

### 권장 역할 분리

- `01_vision_parser`
  - 시각 문서 구조화 전용
- `02_long_doc_analyzer`
  - 장문 문서/섹션 단위 분석
- `03_json_constructor`
  - 규격화된 결과 생성
- `04_search_librarian`
  - 임베딩/RAG 탐색
- `05_security_guard`
  - PHI/민감정보/영업비밀 방어
- `06_main_engine`
  - 메인 판단 및 정밀 검토

핵심 원칙:

- 역할과 모델은 분리한다.
- 워크벤치는 "어떤 모델"이 아니라 "어떤 에이전트 역할"을 호출한다.
- 추후 고사양 GPU로 이식 시, 역할은 유지하고 모델만 상위 버전으로 교체할 수 있어야 한다.

## 8. 보안 및 컴플라이언스

### Zero-Cloud Policy

- 사용자 문서 데이터는 외부 클라우드로 전송되지 않는다.
- 모든 추론, 저장, 인덱싱은 로컬 허브 또는 사내망 안에서 수행한다.

### Air-Gapped Ready

- 인터넷과 단절된 환경에서도 핵심 AI 기능이 유지된다.
- 규제 문서 취급이 엄격한 의료기기 고객사를 고려한 구조다.

### Audit Trail

- AI의 판단 근거를 사용자에게 명시한다.
- 어떤 문서, 어떤 조항, 어떤 참조 근거로 판단했는지 기록한다.
- 사람이 검토하고 승인할 수 있는 explainable workflow를 유지한다.

### Data Handling

- 개인정보 / 민감정보 / 영업비밀 문구에 대한 사전 마스킹 레이어
- 로컬 저장소 암호화 옵션
- 관리자 전용 접근/승인 흐름

## 9. 하드웨어 전략

PikkTob은 "무조건 고가 GPU 장비"보다 **최소 단가 + 실무 효율**을 우선한다.

## 9.1 후보 1: 리퍼비시 초소형 PC

- 예시: HP ProDesk Tiny / Dell Optiplex Micro
- 기준 사양:
  - Intel 8세대 이상 i5
  - RAM 32GB 업그레이드
- 예상 단가:
  - 20만 원 ~ 25만 원

장점:

- 저렴한 도입 비용
- 안정적인 CPU 추론
- 내부 테스트/파일럿용으로 적합

한계:

- 대형 모델 실시간 응답은 어려움
- Pro C는 최적화 또는 오프로딩 전제가 필요

## 9.2 후보 2: NPU 탑재 SBC

- 예시: Orange Pi 5 16GB / 32GB
- 예상 단가:
  - 15만 원 ~ 20만 원

장점:

- 저전력
- 초소형
- 엣지 허브/현장 설치형에 적합

한계:

- 모델 호환성, 드라이버, 안정성 확보가 선행되어야 함

## 9.3 상위 확장형

- 향후 고사양 GPU/NPU 기반 PikkTob Pro 라인업 확장 가능
- 동일한 API/라우팅 계층을 유지하면서 상위 모델을 수용

## 10. 성능 전략

PikkTob은 저사양 클라이언트가 아닌 허브 자원에서 체감 성능을 제공해야 한다.

### 기본 원칙

- 새 모델 로드 전 기존 모델 메모리 해제
- 한 번에 여러 대형 모델 상주 금지
- 요청 유형별 모델 라우팅
- 자주 쓰는 인덱스와 보조 리소스는 상시 유지

### 성능 목표

- 사용자가 "꽂는 순간 빨라졌다"는 체감을 받아야 한다
- 짧은 응답형 태스크:
  - 3초 이내 목표
- 고난도 문서 검토:
  - 수 초 ~ 수십 초 범위 허용
- 중요한 것은 절대 지연보다도:
  - 로컬 PC 부하 감소
  - 안정적 일관성
  - 반복 가능한 문서 처리 흐름

## 11. 배포 및 사업 모델

PikkTob은 초기에는 소프트웨어 단독 운용에서 시작하되, B2B 단계에서 **하드웨어 패키지 구독형**으로 확장하는 전략이 적합하다.

### 단계 1: Software First

- 로컬 소프트웨어 전용
- 파일럿 고객 확보
- 워크벤치 + 로컬 AI 허브 개념 검증

### 단계 2: PikkTob Standard

- 저비용 하드웨어 패키지
- 소규모 팀용
- 네트워크 협업, RAG, 경량 정밀 검토

### 단계 3: PikkTob Premium

- 고사양 자원
- Pro C 정밀 검토
- 다중 사용자, 장문 검토, 강화된 보안 정책

### 비즈니스 포지셔닝

- "월 커피값 수준의 렌탈료로 우리 회사 전용 AI RA를 두세요."
- "문서를 외부로 보내지 않는 사내 전용 AI 허브"
- "저사양 PC 환경에서도 기업급 규제 검토를 호출하는 로컬 AI 워크스테이션"

PikkTob은 단순히 속도를 올리는 보조 장치가 아니다.

- med.piKKto의 AI 기능을 기업 환경에 맞게 물리적으로 분리한다.

즉 PM 관점에서 PikkTob은 다음으로 요약할 수 있다.

> "픽톱은 저비용 하드웨어에서 작동하는 med.piKKto 전용 분산형 AI 워크스테이션이다.  
> 유저 PC가 저사양이어도, 픽톱만 연결하면 로컬 네트워크 안에서 고사양 규제 문서 AI를 안전하게 호출할 수 있다."

## 13. 마일스톤 단계

PikkTob은 처음부터 완성형 하드웨어 제품으로 가지 않고, **로컬 AI 허브 소프트웨어 -> 파일럿 허브 -> 엔터프라이즈 허브 -> 표준/프리미엄 제품** 순으로 확장하는 방식이 가장 현실적이다.

### M0. Feasibility Spike

- 목표:
  - PikkTob 형태의 분리형 AI 허브가 실제로 필요한지 기술적으로 검증
- 범위:
  - 단일 장비에서 FastAPI 기반 추론 브리지 구동
  - med.piKKto 클라이언트와 1회 왕복 호출 연결
  - 경량 메인 모델 1종으로 문서 일부 검토 성공
- 권장 자원:
  - 기존 고사양 테스트 장비 또는 Mac / Tiny PC 임시 장비
- 완료 기준:
  - 문서 일부를 보내고 안정적인 응답을 돌려받음
  - 로컬 PC와 추론 허브 분리 가치가 수치로 확인됨

### M1. PikkTob Hub MVP

- 목표:
  - 1인용 로컬 AI 허브를 med.piKKto와 실제로 결합
- 범위:
  - 단일 사용자 기준 허브 연결
  - LAN 또는 로컬 IP 기반 연결
  - `Search Librarian`, `Evidence Auditor`, `Security Guard` 우선 구현
  - 기본 리소스 모니터와 모델 상태 확인
- 권장 자원:
  - 최소: 16~32GB RAM 기반 Tiny PC
  - 권장: 32GB RAM, 빠른 SSD, 유선 LAN
- 완료 기준:
  - med.piKKto에서 허브를 연결하면 외부 클라우드 없이 검토 요청 수행
  - 사용자 PC가 무거운 추론에서 분리되는 체감이 생김

### M2. Pilot Hub

- 목표:

## 16. 결론

PikkTob은 med.piKKto를 단순 웹 SaaS에서 끝내지 않고, **B2B 엔터프라이즈 온디바이스 AI 플랫폼**으로 확장하는 핵심 인프라다.

이 제품의 성공 조건은 세 가지다.

1. 사용자 PC 스펙과 무관하게 체감 성능을 제공할 것
2. 외부 클라우드 없이도 안전하게 문서를 처리할 것
3. 문서 맥락과 검토 흐름을 이해하는 실무형 AI 허브가 될 것

이 조건을 충족하면 PikkTob은 단순 하드웨어가 아니라, med.piKKto의 기업용 진입 장벽이자 강력한 차별화 수단이 된다.
